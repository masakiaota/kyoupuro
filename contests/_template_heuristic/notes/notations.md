# 記号一覧

このファイルは、問題で使う記号とコード上の代表名・型・制約の正本である。以下は理想的な記載例であり、実際には問題固有の記号で置き換える。(置き換えたらこの文章を削除せよ。)


### 入力サイズ `N`

問題の主要なサイズを表す。グリッド問題なら盤面サイズ、列問題なら列長のように、問題ごとの主サイズとして使う。

- コード上の代表名: `n`
- Rust の想定型: `usize`
- 制約・不変条件: 問題文の制約に従う。0-indexed の添字境界判定の基準になる。
- 備考: 問題に複数サイズがあるなら、`N` を使う対象をこのファイルで明示する。


### prefix 長 `\ell`

いま維持したい先頭一致部分の長さを表す。prefix を段階的に伸ばす議論で使う。

- コード上の代表名: `ell`
- Rust の想定型: `usize`
- 制約・不変条件: `0 <= ell <= target.len()` のように、対象列の長さを超えない。
- 備考: `prefix` という語を使うときは、何の prefix かを併記する。

必要なら数式もこのように `$$` を単独行で前後から囲んで書く。

$$
\text{prefix} = target[0..\ell)
$$
