# Important Properties

このファイルは、問題の重要な性質を整理する場所である。記号の定義・代表名・型・制約は `notes/notations.md` を参照し、ここでは性質そのものだけを書く。以下は仮の記入例であり、実際には問題固有の性質で置き換える。(置き換えたらこの文章を削除せよ。)

## 1. prefix を壊さずに局所探索できる（これは例）

### 主張
先頭 `\ell` 要素を固定したまま suffix 側だけを操作できるなら、探索中に suffix が乱れても prefix の正しさは保てる。

### なぜ成り立つか
操作の影響範囲が suffix に限られており、prefix の要素や順序には触れないからである。したがって、prefix が満たしていた一致条件はそのまま保存される。

### 実装上どう効くか
状態遷移のたびに prefix 全体を再検証せず、suffix だけ差分更新すればよい。局所探索、焼きなまし、ビーム探索で 1 遷移あたりの計算量を減らしやすい。

## 2. スコア差分が局所情報で計算できる（これは例）

### 主張
1 回の遷移で変化する要素が局所的なら、全体スコアを毎回再計算しなくても、変化部分の差分だけでスコア更新できる。

### なぜ成り立つか
遷移前後で不変な部分の寄与は打ち消し合うため、変化したセル、辺、区間などの寄与だけを比較すれば十分だからである。

### 実装上どう効くか
`evaluate()` の全計算を避けて、`apply` / `undo` と一緒に差分更新関数を持てる。反復回数を増やしやすくなり、探索の試行数を確保しやすい。
