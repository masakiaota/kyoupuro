# Ideas

## 実装済み
- `v002_window_dp`
  - 行 snake / 列 snake とその逆順から初期路を選ぶ
  - 長さ 8, 9 の窓ごとに局所 DP で並べ替え、値の大きいマスを後ろへ寄せる
- `v003_two_opt_window`
  - `v002` を土台にして、経路区間の反転による `2-opt` 近傍を追加する
  - `window DP + 2-opt` の局所探索系
- `v004_cycle_relocate`
  - まず盤面全体を巡るハミルトンサイクルを作る
  - サイクルの切断位置を最適化してパス化し、`relocation` と `cycle 2-opt` で改善する
  - 最後に窓 DP で局所調整する
- `v070`
  - `5 x 2` ブロックを単位に帯 DP で大域骨格を作る
  - 盤面の対称変換も試し、最後に `window DP + 2-opt` で仕上げる
  - `帯 DP` と `ブロック分割` を使った本格系

## 未実装
- 順序先行 + 経路修復
  - まず `A` 昇順を理想順序として置き、そこから 8 近傍ハミルトン路へ崩していく
  - 単純 swap ではなく、区間再接続や部分路修復を中心に考える
  - 本命1: ペナルティ連続法 (`score - λ * 違反数`)
    - 小さい `λ` で順序優先の高スコア配置を作り、`λ` を段階的に上げて合法化へ寄せる
  - 本命2: ブレークポイント駆動 Ejection Chain 修復
    - 非合法エッジを起点に区間押し出し連鎖で再接続し、順序損失を抑えて違反を減らす
  - 本命3: Rank-Band 制約つきビーム探索
    - 時刻 `t` に使う候補を理想順位 `t±w` に制限し、詰まり時だけ `w` を広げて構築する
- 両端からのパス構築
  - 片側で低値を前半に集め、もう片側で高値を後半に集める
  - 始点・終点が自由であることを強く使う構成

## メモ
- 既存実装は「合法路を作ってから局所探索」と「帯 DP / ブロック分割」の系統をすでに触っている
- 未着手の本命は「順序先行 + 経路修復」と「両端からのパス構築」である
- 実験知見: 蛇行（snake）系の初期解は平均点を大きく上回りにくく、初期骨格としては弱い
