# AHC063 記号一覧

### 盤面サイズ `N`

正方形グリッドの1辺の長さを表す。

$N$

- プログラム中では `n`（または `input.N`）
- 制約 `8 <= N <= 16`

rust における型は以下の通り

`usize`

---

### 目標色列長 `M`

好みの色列 `d_0..d_{M-1}` の長さを表す。

$M$

- プログラム中では `m`（または `input.M`）
- 制約 `(N^2)/4 <= M <= 3(N^2)/4`

rust における型は以下の通り

`usize`

---

### 色数 `C`

使用する色IDの最大値を表す。

$C$

- プログラム中では `color_count`（または `input.C`）
- 制約 `3 <= C <= 7`

rust における型は以下の通り

`usize`

---

### マス座標 `(i,j)`

盤面上の位置を表す。`i` は行（下方向）、`j` は列（右方向）である。

$(i,j)$

- プログラム中では `(i, j)` / `row, col`
- 制約 `0 <= i, j < N`

rust における型は以下の通り

`(usize, usize)`

---

### 餌マップ値 `f_{i,j}`

マス `(i,j)` の餌の色を表す。`0` は餌なし。

$f_{i,j}$

- プログラム中では `food[i][j]`（または `state.f[i][j]`）
- 制約 `0 <= f_{i,j} <= C`、初期状態で `f_{0,0}=...=f_{4,0}=0`、非ゼロ個数は `M-5`

rust における型は以下の通り

`Vec<Vec<usize>>`

---

### 蛇長 `k`

頭からしっぽまでのセグメント個数を表す。

$k$

- プログラム中では `len`（または `state.ij.len()`）
- 制約 初期値 `k=5`、操作中は問題仕様上 `k <= M`

rust における型は以下の通り

`usize`

---

### 位置インデックス `p`

色列や座標列の添字として使う。

$p$

- プログラム中では `p` / `idx`
- 制約 文脈依存（例: `0 <= p <= k-1`, `5 <= p <= M-1`）

rust における型は以下の通り

`usize`

---

### 一時目標列 `tmp_d`

探索や局所構築のために、その場で定める色列を表す。最終目標列 `d` と一致していてもよいし、segment 構築や在庫戦略の都合で別の列になっていてもよい。

$tmp_d$

- プログラム中では `tmp_d` / `target_colors`
- 制約 各要素は `1 <= tmp_d[p] <= C`

rust における型は以下の通り

`&[u8]`（または `Vec<u8>`）

---

### 確定済みprefix長 `\ell`

一時目標列 `tmp_d` の先頭から、蛇の色列と一致していて維持したい長さを表す（`p` とは別の記号）。

$\ell$

- プログラム中では `ell`（または `confirmed_len`）
- 制約 `0 <= \ell <= |tmp_d|`、確定条件は `c_0..c_{\ell-1}=tmp_d[0..\ell)`

rust における型は以下の通り

`usize`

---

### prefix（保ちたい先頭一致列）

現在の議論で「prefix」と呼ぶものは、一時目標列 `tmp_d` の先頭 `\ell` 個

$$
tmp_d[0..\ell)
$$

を指す。最終目標列 `d` の先頭 `\ell` 個とは限らない。

- プログラム中では `&tmp_d[..ell]` / `prefix_target`
- 典型例:
  - 通常の prefix 伸長では `tmp_d = d`
  - 在庫戦略の segment 構築では `tmp_d = seg`

rust における型は以下の通り

`&[u8]`

---

### 蛇座標列要素 `(i_p,j_p)`

蛇の `p` 番目（頭が `p=0`）の座標を表す。

$(i_p,j_p)$

- プログラム中では `state.ij[p]`
- 制約 `p=0..k-1`、しっぽ以外の座標は互いに異なる

rust における型は以下の通り

`(usize, usize)`（`Vec<(usize, usize)>` の要素）

---

### 蛇色列要素 `c_p`

蛇の `p` 番目セグメントの色を表す。

$c_p$

- プログラム中では `state.c[p]`
- 制約 `1 <= c_p <= C`

rust における型は以下の通り

`usize`（`Vec<usize>` の要素）

---

### 目標色列要素 `d_p`

好みの色列の `p` 番目の色を表す。

$d_p$

- プログラム中では `input.d[p]`
- 制約 `1 <= d_p <= C`、`d_0=...=d_4=1`

rust における型は以下の通り

`usize`（`Vec<usize>` の要素）

---

### 一時目標列要素 `tmp_d_p`

一時目標列 `tmp_d` の `p` 番目の色を表す。

$tmp_d_p$

- プログラム中では `tmp_d[p]`
- 制約 `1 <= tmp_d_p <= C`

rust における型は以下の通り

`u8`（`&[u8]` / `Vec<u8>` の要素）

---

### 移動方向 `d`

各ターンで頭を動かす方向を表す（`d_p` とは別物）。

$d$

- プログラム中では `dir`
- 制約 `d \in \{U,D,L,R\}`、盤外移動とUターンは禁止

rust における型は以下の通り

`char`（または内部表現として `usize` 0..3）

---

### ターン番号 `t`

操作列のインデックスを表す。

$t$

- プログラム中では `turn`
- 制約 `0 <= t <= T-1`

rust における型は以下の通り

`usize`

---

### 出力操作文字 `a_t`

`t` ターン目に出力する移動方向文字を表す。

$a_t$

- プログラム中では `ops[t]`（または `out.out[t]`）
- 制約 `a_t \in \{U,D,L,R\}`

rust における型は以下の通り

`char`（または方向 `usize` に変換して保持）

---

### 移動後座標 `(i'_0,j'_0)` / `(i'_p,j'_p)`

`(i'_0,j'_0)` は移動後の頭座標、`(i'_p,j'_p)` は移動後座標列の要素を表す。

$(i'_0,j'_0), (i'_p,j'_p)$

- プログラム中では `next_head`, `next_pos[p]`
- 制約 `(i'_0,j'_0)` は盤内かつUターンでない

rust における型は以下の通り

`(usize, usize)`（または `Vec<(usize, usize)>`）

---

### 噛みちぎり位置 `h`

頭と重なった胴体のインデックスを表す。

$h$

- プログラム中では `bite_idx`
- 制約 `1 <= h <= k-2`

rust における型は以下の通り

`usize`

---

### 操作回数 `T`

出力した操作列の長さを表す。

$T$

- プログラム中では `ops_len`（または `out.out.len()`）
- 制約 `0 <= T <= 100000`

rust における型は以下の通り

`usize`

---

### 不一致数 `E`

`c_p != d_p` となる位置数（`p=0..k-1`）を表す。

$E$

- プログラム中では `mismatch`
- 制約 `0 <= E <= k`

rust における型は以下の通り

`usize`（最終スコア計算時に `i64` へ変換）

---

### 一様乱数関数 `rand(L,U)`

`L..U` の整数を一様ランダムに返す関数を表す。

$\mathrm{rand}(L,U)$

- プログラム中では `rng.gen_range(L..=U)`
- 制約 `L <= U`

rust における型は以下の通り

引数と戻り値は同じ整数型（例: `usize`）

---

### 累積区切り点 `n_c`

入力生成で色個数を作るための補助列を表す。

$n_c$

- プログラム中では `n[c]`
- 制約 `n_0=0`, `n_C=M-5-C`、`n_1..n_{C-1}` は `0..M-5-C` から生成して昇順化

rust における型は以下の通り

`usize`（`Vec<usize>` の要素）

---

### 色別個数 `m_c`

色 `c` の個数を表し、`m_c=n_c-n_{c-1}+1` で定義する。

$m_c$

- プログラム中では `m[c]` / `color_freq[c]`
- 制約 `m_c > (M-5)/2` となる色がある場合は再生成

rust における型は以下の通り

`usize`（`Vec<usize>` の要素）

---

### 配置色列 `d'_p`

`d_5..d_{M-1}` をシャッフルした盤面配置用の色列を表す。

$d'_p$

- プログラム中では `d2[p]` / `shuffled_target[p]`
- 制約 `p=5..M-1`、`d_5..d_{M-1}` の順列

rust における型は以下の通り

`usize`（`Vec<usize>` の要素）

---

### 絶対スコア `abs_score`

最終状態に対する絶対評価値を表す。

$$
\mathrm{abs\_score}=T + 10000 \times \left(E + 2(M-k)\right)
$$

- プログラム中では `abs_score`（計算式は `turn as i64 + 10000 * (mismatch + 2 * (m as i64 - k as i64))`）
- 制約 小さいほど良い（最小化）

rust における型は以下の通り

`i64`

---
