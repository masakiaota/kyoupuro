# Ideas

## 1. 固定prefix伸長戦略（`ℓ+1`）

### 概要

基本方針は、先頭一致長 `ℓ` を 1 ずつ伸ばすことである。  
具体的には、状態 `S` が

`c_0..c_{ℓ-1} = d_0..d_{ℓ-1}`

を満たすとき、次に `ℓ+1` まで一致させる操作列を探す。

この戦略は、`v005_color_goal.rs` の中心思想である。

### 成立仮定（アイデアの核）

次が常に成り立つと仮定する。

- 任意の盤面状態 `S` と `ℓ`（`5 <= ℓ < M`）について、
- `S` が先頭 `ℓ` 一致を満たしていれば、
- 先頭 `ℓ+1` 一致へ遷移できる操作列 `π` が存在する。

この仮定が真なら、帰納的に `ℓ=5` から `ℓ=M` まで到達でき、  
結果として全餌を色順どおりに食べ切れる。

### 実装上の読み替え（`v005_color_goal.rs`）

- `ell` を `5` から開始し、`extend_one(ell)` を繰り返す。
- `extend_one` は目標色 `d[ell]` を 1 個食べて `prefix` を `ell+1` へ伸ばす操作を探索する。
- 探索は段階的に緩和する。
1. safe 段階: 非目標食いなし・噛みちぎりなし。
2. rescue 段階: 非目標食いや噛みちぎりを制限付きで許容。
3. フォールバック: 個別ターゲットに対して移動・縮退・最終取得を試行。

### 既知の問題

現状実装はこの仮定を常に実現できていない。  
`ℓ` が十分長くなると、`+1` 操作が見つからず停止するケースがある。

主な失敗要因候補:

1. 盤面占有が増えて有効経路が細り、探索深さ/ノード上限内で解が見つからない。
2. 一時的な非目標取得や噛みちぎりの回復経路が局所的に閉じる。
3. `+1` 自体は存在しても、現在の探索順序・評価では到達できない。

### 位置づけ

この章は「固定prefixを守りながら伸ばす」基準戦略である。  
今後の派生戦略（運び屋戦法、戦略的prefix切断）は、この章の失敗ケースを回避するための拡張として扱う。

## 2. 運び屋戦法（未実装）

### 概要

`d` の部分列をその場で最終連結せず、いったん蛇上で仮組みし、  
回収しやすい地点へ運んで噛みちぎり、盤面在庫として蓄積する。

この「仮組み -> 搬送 -> 在庫化」を複数回行い、終盤に在庫を一気に回収して最終列を構築する。

### 基本操作

1. 部分列仮組み  
   対象区間 `d[L..R]` に対応する色列を、通常の `ℓ+1` 的な手順で蛇の suffix 側に構成する。
2. 搬送  
   仮組み済み部分列を保持したまま、後で回収しやすい保管点へ移動する。
3. 戦略的噛みちぎり（在庫化）  
   自己接触で噛みちぎり、仮組み部分を盤面上の餌列として落とす（stock 化）。
4. 反復  
   別区間でも同様に実行し、複数の部分列在庫を作る。
5. 終盤一括回収  
   在庫の回収順を制御し、最終的な `d` の連結を一気に完成させる。

### 狙い

- 蛇長を伸ばし続けずに、盤面上の順序情報を外部化できる。
- `ℓ+1` 直列伸長で詰まりやすい中盤以降を、在庫戦略で分割統治できる。
- 終盤は「回収順制御問題」に還元でき、局所探索の自由度が上がる。

### 既知の設計課題

1. 保管点設計  
   在庫同士が干渉せず、かつ回収導線が短い配置規則を定める必要がある。
2. 切断点設計  
   どの `h` で噛みちぎると在庫形状と復帰コストのバランスが良いかを定式化する必要がある。
3. 回収順最適化  
   終盤の回収順が手数と失敗率を大きく左右するため、順序決定ロジックが必要である。
4. 評価軸  
   平均絶対スコアだけでなく、ケース別順位への寄与（特に簡単ケースの手数）を評価に含める必要がある。

### 位置づけ

この戦法は、固定prefix伸長戦略の「`ℓ` が大きくなると `+1` が見つからない」問題に対する  
構造的な回避策として位置づける。

## 3. 在庫戦略のエッセンス

### 概要

目標列 `d[5..]` をその場で最後まで蛇の体内に保持するのではなく、  
いったん盤面上の「在庫」として外出しし、最後にまとめて回収する。

具体的には、`d[5..]` を後ろから長さ `N-2` ごとの segment に分割し、  
各 segment を短い蛇で 1 本ずつ構築して、列ごとの縦在庫として保管する。

### 核となる考え方

1. 目標列の分割
   `d[5..]` を後ろから小 block に切り、直接 1 本の長い蛇として育てない。
2. 在庫化
   各 block をいったん盤面上へ順序付きで落とし、色列情報を蛇の外へ退避する。
3. 短蛇運用
   在庫化のたびに蛇を短い carrier に戻し、次の block 構築も短蛇で行う。
4. 領域封鎖
   一度保管した左側列は再訪せず、活動領域を右へ単調に押し出す。
5. 終盤一括回収
   列の向きを偶奇で反転させておき、最後は右から左へ蛇行回収するだけで正順につなぐ。

### この戦略が狙うもの

- 長蛇のまま探索して小回りを失う問題を避ける。
- 探索の難しさを「短蛇で次の 1 block を運ぶ」問題へ分解する。
- 終盤を探索問題ではなく、ほぼ幾何的な決め打ち回収問題へ変換する。

### 位置づけ

この戦略の本質は、  
「目標列を prefix として逐次伸ばす」のではなく、  
「目標列を小 block に分けて盤面在庫として再配置し、最後に一括回収する」  
という点にある。
