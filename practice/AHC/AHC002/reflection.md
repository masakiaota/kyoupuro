# Reflection

## 反省メモ

1. 初動で「単純で強い初期解」を最優先すべきだった  
今回の問題は、複雑な評価関数付きBFSよりも、DFSで長い初期解を作る方が自然だった。次回は「まず最小実装の強いベースラインを作る」を明示してAIに指示し、方針をぶらさない。

2. 問題を時系列として見すぎない  
見た目は逐次遷移でも、`start` と `end` を固定した時点で向きの意味が薄れる構造がある。この種の「実は時系列でない問題」を早期に見抜けるよう、状態の対称性・不変量を先に確認する。

3. 焼きなましの巨大近傍（破壊再構築）を型として覚える  
「どこを壊すか」「どう再構築するか」「どう受理するか」をセットで設計する。今回の改善はこの型がはまった結果なので、今後も最初から候補に入れる。

4. SAの式変形は探索中の枝刈りに使える  
`u < exp(Δ/T)` を `Δ > T ln u` に変形すると、受理閾値を先に作れる。これにより近傍構築中でも「この先どう頑張っても受理不可」な枝を切れる。このテクニックを再利用可能な知識として定着させる。
